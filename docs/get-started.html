Structure & design

The layers are in separated SASS files collected in the structure and theme folders. Each structure file has a matching theme file and includes it with the @import rule. There's a Grunt task which is responsible for merging these two layers into one CSS file. For instance dog.scss that contains the structural part and imports _dog.scss with the design layer compiles into dog.css.

The unit variables are stored in the _global-variables.scss which is equally used by both layers. The colors are stored in the _colors.scss that is only accessible by the the design layer. Therefor you can easily redesign all of your UI elements through these two config files.

Pick your module

suit has four folders as building block categories: The essential folder contains the core modules like the reset.css or the grid system. The plugin folder collects all the components you need to build a simple application: icons, tables, form components etc... Extensions are here to upgrade existing modules: for instance you can easily merge the essential button module and icon plugin with button-pill.css thus you can place icons into buttons by adding a single class and the icon itself. The forth level is the app folder which is the place for "organisms" (as Brad Frost calls it in his Atomic Design methodology) like the above mentioned styleguide or a complex header with search an lists.
