About me

Projects and experience

Suit

Ongoing personal experiment

Based on the ups and downs of Snorky and Ustream's huge amount of redundant static code I started to play with my very own modular theme approach. My problem with the existing frameworks like Bootstrap and Foundation is that they are barely customizable without duplicating the codebase making the amount of required static asset quite large.

In Suit's SASS based setup there's a structural and a design layer for each and every module. The core part resets, sets up and positions the elements while the theme part handles the design related CSS rules from gutters to block heights, colors and border sizes declared in a config file so can create custom themed elements without touching the core properties and reduce the divergence among UI fragments.

Suit includes Gulp tasks for SASS lint and CSS minify configured app-wisely therefore you can build up your pages and apps based on the element requirements. Feel free to take a peek at it on Github.
