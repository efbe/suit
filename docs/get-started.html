Structure & design

The layers are in separated SASS files collected in the structure and theme folders. Each structure file has a matching theme file and includes its main components with the @import rule. A Grunt task is responsible for merging these two layers into one CSS file. For instance dog.scss that contains the structural part and imports _dog.scss with the design layer compiles into dog.css.

The unit variables are stored in the _global-variables.scss which is equally used by both layers. Color variables are stored in the _colors.scss that is only accessible by the the design layer. You can easily customize all of your UI elements by editing these two config files.

Detachable styles

I kept modularity in mind when setting up the design layers, so each and every standalone styling has its own mixin included by the structure file.
Let's take a look at the button module:

After setting up the structural rules it includes the button mixins one by one style-wise.

@import '../../theme/tron/essential/_button';

.button {
	box-sizing: border-box;
	display: inline-block;
	text-align: center;
	text-decoration: none;
	cursor: pointer;

	// webkit reset
	// scss-lint:disable VendorPrefix
	-webkit-appearance: none;

	@include button();
	@include button-disabled();
	@include button-sizes();
	@include button-colors();
	@include button-link();
	...

While the relevant code snippet of _button.scss looks something like this:

// Disabled

@mixin button-disabled {
	&[disabled],
	&.disabled {
		opacity: $disabled-percentage;
		cursor: not-allowed;
		pointer-events: none;

		@include user-select(none);
	}
}


// Sizes

@mixin button-sizes {
	&.small {
		padding-right: $small-padding;
		padding-left: $small-padding;
		...

So if you want to ged rid of any of the unused button versions you just have to remove the inclusion from button.scss and the mixin from _button.scss.

Pick your module

suit has four folders as building block categories:

  • The essential folder contains the core modules like the reset.css or the grid system.
  • The plugin folder collects all the components you need to build a simple application: icons, tables, form components, etc...
  • Extensions are meant to upgrade or connect existing modules: for instance you can easily merge the essential button module and icon plugin with button-icon.css thus you can place icons into buttons by adding a single class and the icon itself.
  • The forth level is the app folder which is the place for "organisms" (as Brad Frost calls it in his Atomic Design methodology) like the above mentioned styleguide or a complex header with search field and menu items.
It's all about the config

Instead of listing your modules one by one in the head of your document or using an adapter CSS file which imports them, suit has a nice Grunt task which collects your modules and minifies them into one cacheable CSS file. These so-called groups can be found in the dist/css/config folder and you can create as many as you want according to your module usage.
I've created some examples like the essential group which only fetches the files from suit's essential UI component folder. Let's take a peek:

The CSS config file collects the required files with the @import rule.

/*
 * essential.css - essential module collector config file
 * see GruntFile.js / cssmin() for more info
 * @output essential.min.css
 */

@import '../generated/essential/reset.css';
@import '../generated/essential/typo.css';
@import '../generated/essential/utilities.css';
@import '../generated/essential/grid.css';
@import '../generated/essential/button.css';

The Grunt task creates your minified CSS group according to your configuration. 

// Minify CSS files according to [config]/*.css
// @info https://github.com/gruntjs/grunt-contrib-cssmin

cssmin: {
	target: {
		options: {
			restructuring: false, // keep the grouped files in cascading order
			shorthandCompacting: false,  // no need to messing up my ruleset
			roundingPrecision: -1,
			processImport: true // allows you to import from an external source
		},

		// Minified packages come here!
		files: {
			'dist/min/essential.min.css': [ // minified output
				'dist/css/config/essential.css' // reference to your config file
			],
			// Example for merging already existing groups
			'dist/min/styleguide.min.css': [
				'dist/css/config/essential.css',
				'dist/css/config/styleguide.css'
			]
		}
	}
}

This way you'll get the essential.min.css file containing all of your hand-picked modules cascading and perfectly minified.
